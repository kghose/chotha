About

Chotha is a program to store and organize notes and citations for, primarily, academic research.

Usage

Most operations are self explanatory, especially with the help in the popup text. Only the more non-intuitive operations are described here:

Links between items

You can make links to another note or source. To make a link to another note, in the text field type [text to link][note:XXX] where XXX is the note id. To make a link to a source type in [:citekey] where citekey is the citekey of the source (e.g. savitzky1964)

Historical note: Chotha also lets you use [source:citekey] to indicate sources, but this is deprecated and tags won't work with this old form

Creating a source

Type in a pubmed id, a DOI or any other pubmed query into the title box (where you would otherwise type in the title of the note you are creating). Check the 'this is a paper' checkbox right below that. Type in your notes about the paper and your keywords as usual.

When you click save, Chotha will save your note as usual, and then will go to pubmed, fetch the citation data, create a new citation and link your note to it.

It will then display a citation editing page, where you can review the citation information. The citation editing box accepts entries formatted in BibTeX. You can edit or paste BibTeX (e.g. from Google Scholar) into this box.

When you click save, the citation information is saved.

If you leave the title blank then a blank citation from will be returned

The program actually fetches the first hit returned by the query. If you type a pubmed id or a DOI then there is only one hit. If you type a title or some other search term, chances are there will be several of which you will get only the first.

Chotha is not meant for doing searches on pubmed - only for retrieving targeted individual papers at a time.

Source management

Organising sources into a bibliography

A bibliography is simply a collection of sources in a note (inserted by typing [:citekey]).

You could simply order the sources alphabetically or any other way in a note, perhaps by writing a narrative (as if you were writing a paper) and insert citations as you build the text. Chotha is not optimal in this later case, since the text editor is very primitive and Chotha uses plain text as its basic storage form.

A more convenient way of arranging a bibliography is to use tags after the citation

[:schultz1992][t:tag1, tag2, tag3]
So, to build up a bibliography of Basal Ganglia function you would do something like
[:hasegawa2004][t:dlPFC, principal sulcus, objects] - My brief note about this paper
[:battig1960][t: caudate, dlPFC]

When Chotha detects these tags it will build up an alphabetical list of them which it will present at the beginning of the displayed note. This followed by the sources being listed under each tag appropriately such that clicking on a tag will show you the list of relevant sources. The actual note you type will be available under this sectioned list.

Exporting sources in a bibliography

In the show note page there is a link that says 'export sources'. If you have added any sources to the note then clicking this link will let you export the sources to a file of your choice in a format of your choice.

Somewhere in the note, on a new line type

File:
followed by the name of the file you want the sources exported to, followed by a newline.

In another newline type

Mode:
followed by one of the words word, bibtex, ris

word will export the sources to MS Word's XML format which you can then add to your document through MS Word's own bibliography system.

bibtex will export to a BibTeX file which you can use with LaTeX + BibTeX

ris will export to the ris format which can be imported by EndNote and other reference manager programs.

So:
File:/Users/kghose/Research/references.bib
Mode:bibtex
will save all the citations in the note to references.bib in bibtex format

Advanced source search

If you know SQL you can type in a WHERE clause in the search box instead of a word query to find sources. Begin the query with :s followed by a space and then the SQL WHERE clause.

For example the following search
:s journal LIKE 'Journal of Neuroscience'
will return all sources whose journal name is "Journal of Neuroscience".

Note that this will execute the SQL you type directly on your database, so becareful what you type.

Formatting notes

Chotha uses 'markdown' to allow you to format notes. Formatting cheat sheet

Math text

Chotha uses Peter Jipsen's ASCIIMathML to convert latex strings to equations. Type equations between dollar signs '$' to display your text as math when you view it.

The unauthorized biography

The name: Chotha (চোথা) is a Bengali slang word for notes that I picked up when I was at Jadavpur University. I believe it is general college slang for lecture notes or notes of any kind about a topic. Even my mother knew the word so I guess it is a fairly polite word that has been in use for some time.

The origins of Chotha were probably the little hanDbase database I created on my handspring visor (which somebody stole from my office at A.V. Williams building grr.) This was replaced by R-A which I wrote in C++ and QT and which used MySQl.

I then learned about Rails and migrated R-A to RRiki (written in Ruby, Rails and which used MySQL) as a way to procrastinate on writing my thesis.

Then I decided it was getting confusing, in my old age, to write code in Ruby (This stuff) AND Python (work work) and I was getting ever more infatuated with Python, and getting disgruntled by the complexity of Rails and the upgrading and so on.

Finally, I found Bottle, and over the course of a few weeks, split off the Diary sub-app and the the main RRiki app into two Python programs that used the Bottle framework. Chotha is RRiki with the hierarchical keywords replaced by conjunction keywords and with all that fancy AJAX crap removed. The program that spanned oh, I don't know, perhaps 40 files, now uses perhaps three files that I actually like maintaining and looking at.

Rails had its place, but for my simple single user, desktop requirements, I think I will stick to micro-frameworks, and python + Bottle is just awesome for that.
